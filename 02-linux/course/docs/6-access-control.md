### Контроль доступа

```
Пользователь (u)
Группа пользователей (g)
Все остальные (o)
Все (a)
```

1. Например 
```
rwxrw-r--
rwx: Права пользователя
rw-: Права группы
r--: Права для всех остальных
=====================================
r: read
w: write
x: execute
====================================
rwx rw- r-- это как => 111 110 100 => 111(7) 110(6) 100(4)
chmod 0777 
chmod 764 
```

2. Например дать права доступа к файлу
```
$ ls -la | grep file
-rw-rw-r-- 1 yao yao    5 апр 16  2023 file1
-rw-rw-r-- 1 yao yao    5 апр 16  2023 file1_hard_link
-rw-rw-r-- 1 yao yao    5 апр 16  2023 file1_soft_link
-rw-rw-r-- 1 yao yao    0 апр 16  2023 file2
-rw-rw-r-- 1 yao yao    0 апр 16  2023 file3
-rw-rw-r-- 1 yao yao    0 апр 16  2023 file4
-rw-rw-r-- 1 yao yao    0 апр 16  2023 file66
-rw-rw-r-- 1 yao yao 1018 апр 16  2023 file.txt
-rw-rw-r-- 1 yao yao    0 апр 16  2023 .hidden_file
$ chmod 664 file66
```


3. Например дать права доступа к файлу другой способ 
```
Example
$ chmod u+x file66                   (добавляем права у пользователя на исполнение файла)
$ chmod u+x composer_from_docker.sh  (добавляем права у пользователя на исполнение файла)
$ chmod g-w file66                   (отбераем все группы которые есть прова на записи)
```


4. Изменение павил доступа 
```
$ chmod 744 file
$ chmod g+w file (все в группе получают право на запись)
$ chmod o+x file (все кроме пользователя и группы лишаются прав на выполнение файла)
$ chmod o-x file (все кроме пользователя и группы лишаются прав на выполнение файла)
```
